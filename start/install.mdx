Neroli is a Mac app first. Install it, connect the agent tools you already use,
then open a repository and start with one small task.

## What you need

- macOS 15.6 or later.
- A Git repository on your Mac.
- At least one supported coding agent, such as Claude Code or Codex.
- GitHub authentication if your workflow includes pull requests or GitHub-backed actions.

<Note>
  You do not need every provider on day one. Start with the agent you trust, then add more when parallel work is useful.
</Note>

## Install the app

<Steps>
  <Step title="Download Neroli">
    Get the latest Neroli build from the website or your team distribution channel.
  </Step>

  <Step title="Move it to Applications">
    Drag `Neroli.app` into `/Applications`.
  </Step>

  <Step title="Open Neroli">
    Launch the app and approve the macOS prompts needed for your workflow.
  </Step>

  <Step title="Open a repository">
    Choose a local Git repository. Neroli turns it into a workspace for sessions, branches, panes, and review state.
  </Step>
</Steps>

## Connect your agent

Neroli coordinates the agent tools installed on your Mac or configured for your
account. If a provider is not available inside Neroli, sign in to that provider
from your normal terminal, then restart or create a new session.

Common checks:

```sh
git --version
gh auth status
claude /login
codex login
```

## What the CLI is for

Neroli also installs a `neroli` command, but most users do not start there. The
CLI is mainly for agents, repo actions, and advanced debugging. It lets a
session inspect the correct browser pane, simulator, session path, and action
state without guessing from the outside.
