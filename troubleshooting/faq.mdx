## Is Neroli replacing my agent?

No. Neroli coordinates the coding agents you already use. It keeps their work
separate, visible, and reviewable.

## Is a worktree the same as a clone?

No. A worktree is a separate checkout that shares Git object storage with the
repository. It is lighter than a full clone but still has its own files on disk.

## Can two sessions use the same branch?

Usually no. Git normally allows a branch to be checked out in only one worktree
at a time. Use a new branch for a second attempt.

## Why did a command run in the wrong place?

Check the current directory. Commands should run from the session that owns the work.

```sh
pwd
git branch --show-current
neroli identify --json
```

## Should I run multiple agents at once?

Only when the work can be reviewed independently or when the agents are cooperating inside one branch with clear roles.

## Where should repeatable commands live?

Put repo-owned actions under `.neroli/automations`. Keep one-off debugging in terminal panes.
