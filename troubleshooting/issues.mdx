---
title: "Troubleshooting issues"
description: "Fix common setup, provider, branch, and verification problems."
---

Start with the session that owns the problem. Most confusing failures come from
checking the wrong branch, server, browser pane, or simulator.

## A provider is unavailable

Check that the provider CLI is installed and authenticated:

```sh
claude /login
codex login
```

Then restart the session or open a new terminal from the same session.

## Git status looks wrong

Confirm the session path and branch:

```sh
pwd
git status
git worktree list
```

If you are in the main checkout, switch to the session path before running checks.

## A repo action fails

Run the script manually from the session path:

```sh
cd "$NEROLI_WORKTREE_PATH"
.neroli/automations/verify/run.sh
```

Look for assumptions about absolute paths, missing dependencies, or interactive prompts.

## Browser pane shows stale content

Restart the project run command, then reload the browser pane. If the page is
cached at the edge, test with a no-cache request or clear the relevant cache.

## Cloud handoff fails

Read the visible status message first. Common causes include expired auth,
missing runtime capacity, unsupported provider state, or workspace hydration failure.

<Tip>
  The fastest useful bug report includes the session branch, failing command, visible error, and whether the same command works outside Neroli.
</Tip>
