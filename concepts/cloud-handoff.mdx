Neroli is local-first. Cloud handoff is for the moments when a supported session
should keep running away from your Mac and come back with enough state to review.

## Use cloud handoff when

- A longer supported task should keep running while your Mac is unavailable.
- The work does not depend on local-only devices, files, or manual intervention.
- You are comfortable reviewing the returned diff before merging.

## Stay local when

- The task needs a local simulator, device, browser flow, or private file.
- You expect to intervene every few minutes.
- The repository depends on secrets or local state that should not move.

## What stays true

- You still review the resulting diff before merging.
- Local secrets should not be committed or treated as portable state.
- Status in the app should tell you whether the session is local, cloud, synced, or needs attention.
- Handoff support depends on your account, provider, and runtime configuration.

<Warning>
  Cloud support depends on account, provider, and runtime configuration. If a handoff fails, fix the visible issue before retrying.
</Warning>
