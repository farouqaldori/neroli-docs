title: "Testing"

---

Every worktree can have its own dependencies, processes, and local state. Run checks from the same worktree that contains the changes you are reviewing.

## Testing paths

<Tabs>
  <Tab title="Automation">
    Use a `Verify`, `Build`, or project-specific automation when the command should be repeatable and visible in the Neroli UI.
  </Tab>

  <Tab title="Terminal">
    Use a terminal pane for ad hoc commands, debugging, or one-off checks.
  </Tab>

  <Tab title="Browser">
    Use a browser pane when the change needs visual or interaction verification.
  </Tab>

  <Tab title="iOS simulator">
    Use the simulator pane and `neroli ios-simulator` commands for iOS UI verification.
  </Tab>
</Tabs>

## Recommended checks

| Project type | Example check |
| --- | --- |
| Swift macOS app | `cd macos && scripts/build.sh` |
| Backend | `cd backend && npm run check` |
| Website | `cd website && npm run check` |
| Docs | `mint dev` or Mintlify validation if available |

## What to record

Ask the session to report:

- Which commands it ran.
- Whether they passed.
- Any warnings or skipped checks.
- What still needs human review.

<Info>
  Passing tests are not a substitute for diff review. They only lower the risk of accepting the change.
</Info>
