Neroli is designed around one loop: start focused work, observe the evidence,
verify the result, and decide what ships.

## The default loop

<Steps>
  <Step title="Start from a repository">
    Open a Git repository in Neroli.
  </Step>

  <Step title="Create a session">
    Create a session for one task and choose the agent provider you want to use.
  </Step>

  <Step title="Add context">
    Attach files, screenshots, logs, or instructions that define the task.
  </Step>

  <Step title="Observe the work">
    Watch terminal output, browser state, simulator state, and changed files while the session runs.
  </Step>

  <Step title="Verify">
    Run the relevant action, browser check, simulator check, or terminal command from that session.
  </Step>

  <Step title="Review and decide">
    Inspect the diff. Merge, open a pull request, continue the session, or archive it.
  </Step>
</Steps>

## Local and cloud work

<Tabs>
  <Tab title="Local">
    Local sessions run on your Mac. Use them for quick work, UI inspection, and local tools.
  </Tab>

  <Tab title="Cloud">
    Cloud handoff is for supported sessions that should keep running away from your Mac. Neroli syncs the result back for local review.
  </Tab>
</Tabs>

## What to watch

- **Plan:** the intended path before the agent writes too much code.
- **Terminal output:** build, test, and agent logs.
- **Changed files:** the actual Git diff for the session branch.
- **Panes:** browser, simulator, and other surfaces attached to the session.
- **Status:** whether the session is running, waiting, failed, or ready for review.

## Completion criteria

A session is ready to merge only when:

- The diff matches the requested task.
- Verification passed or the remaining risk is understood.
- There are no accidental unrelated files.
- Any local-only files or generated artifacts are excluded.

<Tip>
  Ask the agent to summarize what changed and how it verified the work. Then confirm against the diff yourself.
</Tip>
