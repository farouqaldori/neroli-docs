Neroli can attach an iOS simulator pane to a session. Use it when a mobile
change needs visual inspection, screenshots, accessibility state, or touch input
from the same branch that built the app.

## Recommended flow

<Steps>
  <Step title="Build from the session">
    Use the repository's simulator build script from the current session.
  </Step>

  <Step title="Install and launch in Neroli">
    Install onto this workspace's simulator through Neroli, then launch the bundle.
  </Step>

  <Step title="Inspect before tapping">
    Capture a screenshot or accessibility summary before interacting with the UI.
  </Step>

  <Step title="Record the final state">
    Keep the final screenshot, UI description, or known limitation with the session review.
  </Step>
</Steps>

## Agent CLI examples

```sh
neroli ios-simulator status
neroli ios-simulator screenshot
neroli ios-simulator describe-ui
neroli ios-simulator tap 0.5 0.7
```

<Info>
  Each Neroli workspace owns its active simulator target. Do not pick random devices from `xcrun simctl list` inside Neroli.
</Info>
