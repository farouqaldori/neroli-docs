Run many AI coding agents in parallel. Trust what ships.

AI coding agents make it easy to start more work than you can comfortably
review. Neroli is the Mac app that keeps that work understandable.

Use Neroli when you want to run Claude Code, Codex, or another coding agent
without losing track of the branch, the visible app state, the verification, or
the final merge decision. It gives every task a clear lane from prompt to
review.

## Why Neroli exists

Agent work is only valuable when you can trust the result. Without a coordinator,
parallel sessions quickly become hard to reason about:

- One checkout fills with unrelated edits.
- Terminals, servers, browser tabs, and simulator state drift away from the task.
- A finished agent still leaves you asking what actually changed.
- Running several agents creates more review burden than confidence.

Neroli makes agent work reviewable. Each session has a branch, visible status,
attached panes, and a diff you can inspect before anything reaches your main
branch.

## The user journey

<Steps>
  <Step title="Start with one task">
    Open a repository and start a focused session for one fix, feature, test, or investigation.
  </Step>

  <Step title="Let Neroli keep it separate">
    Neroli gives the session its own branch, files, panes, and review context so it does not collide with other work.
  </Step>

  <Step title="Watch what matters">
    Follow the plan, status, terminal output, browser state, simulator state, and changed files from one place.
  </Step>

  <Step title="Review before you trust it">
    Inspect the diff, run the right verification, compare competing sessions, then merge or archive.
  </Step>
</Steps>

## What Neroli handles

- A calm overview of open sessions, current status, and review state.
- Visible plans, so you can redirect work before the agent spends time in the wrong direction.
- Isolated branches and files so parallel sessions do not trample one checkout.
- Panes that stay with the task: terminal, browser, simulator, logs, and diff.
- Review context such as summaries, line attribution, and conflict surfacing.
- Repo-owned actions for repeatable setup, run, build, verify, and cleanup.
- Local-first operation, with cloud handoff only when you intentionally use it.

<Info>
  You do not need to learn the internals first. Start with one session, review it, then add parallel work when the review boundary is clear.
</Info>
