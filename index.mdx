Neroli is a Mac app for managing AI coding work that would otherwise sprawl
across terminals, branches, browser tabs, and half-finished diffs.

It gives every task a lane: a branch, a worktree, panes, actions, status, cloud
state when needed, and a review decision at the end.

## The problem

Agents are fast at making changes. The hard part is keeping the changes
reviewable.

Without Neroli:

- one checkout gets polluted by unrelated tasks
- the server you are looking at may not match the branch you are reviewing
- new sessions need the same setup steps over and over
- long tasks die when your laptop is closed or busy
- a finished agent still leaves you asking what actually changed

Neroli fixes that by making each agent session a small, inspectable workflow.

<Steps>
  <Step title="Start a focused session">
    Pick one bug, feature, test, or investigation. Neroli gives it its own branch and worktree.
  </Step>

  <Step title="Make it runnable">
    Repository actions can set up dependencies, start the app, build, verify, and clean up from the right session.
  </Step>

  <Step title="Move long work to cloud">
    Supported sessions can keep running in Neroli Cloud, then return to your Mac for review.
  </Step>

  <Step title="Review the proposal">
    Compare the plan, commit story, diff, browser or simulator evidence, and verification result before merging.
  </Step>
</Steps>

## What matters

- **Confidence:** every result is a proposal until you inspect and verify it.
- **Flow:** new sessions can start with setup and run actions already waiting.
- **Parallelism:** split independent work without mixing files or panes.
- **Continuity:** use cloud when the task should keep running away from your Mac.
- **Control:** your main branch only changes after review.

<Info>
  Neroli is local-first. Cloud is an explicit choice for supported sessions, not the default path for every task.
</Info>
