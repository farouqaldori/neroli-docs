Neroli is a developer tool. It runs local commands and agent providers against your repositories, so it should be treated with the same care as a terminal.

## What isolation means

Worktrees isolate development state:

- Separate directories.
- Separate branches.
- Separate running commands.
- Separate review surfaces.

They do not isolate macOS user permissions. A local command can access what your user account can access unless the command or provider adds its own restrictions.

## Secrets

Do not commit secrets. Keep them in local environment files, Keychain, provider login state, or your team's secret manager.

Common files to protect:

- `.env`
- `.env.local`
- local certificates
- API key files
- generated credentials

## Agent changes

Review agent output before merging. Pay special attention to:

- New scripts.
- Permission changes.
- Dependency changes.
- CI or deployment configuration.
- Files outside the requested task.

<Warning>
  Do not run destructive commands from an agent session unless you understand the command and the target worktree.
</Warning>
